# 团队状态报告模板

调度者向 team-lead 或用户报告进度时使用的结构化模板。

## 完整报告模板

```markdown
## 📊 整体进度

| 指标 | 状态 |
|------|------|
| **阶段** | {当前阶段，如: dev / qa / done} |
| **任务完成** | {完成数}/{总数} ✅ {百分比}% |
| **代码提交** | {最新 commit hash / 未提交 / 已推送} |
| **阻塞问题** | {数量，如: 0 / 1 / 2} |

---

### 👥 团队成员

| 成员 | 角色 | 状态 |
|------|------|------|
| {name} | {role} | {status} |

---

### ✅ 任务/Bug 状态

| ID | 描述 | 负责人 | 状态 |
|----|------|--------|------|
| {id} | {desc} | {owner} | {status} |

---

### 📝 最近更新

- {时间}: {事件描述}
- {时间}: {事件描述}

---

### ⚠️ 阻塞问题

{无 / 详细列表}

| 时间 | 问题 | 阻塞者 | 依赖方 | 状态 |
|------|------|--------|--------|------|
| {time} | {desc} | {who} | {depends} | {status} |

---

### 📦 产出物清单

- [{x}] {产出物名称}

---

### 💡 结论/建议

{下一步建议或需要决策的事项}
```

---

## 简洁报告模板（快速同步）

```markdown
## 📊 {团队名} 状态速报

**阶段**: {stage} | **进度**: {x}/{y} ✅ | **阻塞**: {n}

### 今日进展
- {进展1}
- {进展2}

### 明日计划
- {计划1}

### 需要支持
{无 / 具体问题}
```

---

## Bug修复专用报告

```markdown
## ✅ {团队名} Bug修复报告

### 📊 整体进度
| 指标 | 状态 |
|------|------|
| **阶段** | {stage} |
| **Bug修复** | {完成数}/5 ✅ {百分比}% |
| **代码提交** | {commit} 已{pushed} |

### ✅ Bug修复状态
| Bug | 描述 | 模块 | 负责人 | 状态 | 提交文件 |
|-----|------|------|--------|------|----------|
| #1 | {desc} | {module} | {owner} | ✅ | {files} |
| #2 | {desc} | {module} | {owner} | ✅ | {files} |

### 📝 代码提交
```
{commit hash} {message}
```

### 结论
{已完成 / 还需XX}
```

---

## 使用建议

1. **首次报告**: 使用完整模板，建立基线
2. **每日同步**: 使用简洁模板，突出重点
3. **完成报告**: 使用专用模板（如 Bug修复），强调产出物
4. **问题升级**: 详细描述阻塞问题，提出明确的支持需求
